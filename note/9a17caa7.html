setState 异步特性

基于性能考虑，React 通常是批量合并更新，调用 setState() 之后，this.state 并没有马上修改，而是创建了一个中间态作为过渡。

但是有些例外情况，它是同步执行的，比如：eventListeners，Ajax，setTimeout 等。
原因是这些 JS 原生的 API 不在 React 的上下文控制范围，无法进行优化。

如何解决？
